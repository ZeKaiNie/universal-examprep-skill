# exam-cram — zh 学生侧文案包

> 本文件是学生可见文案的 zh 语言包；行为逻辑在 [skills/exam-cram/SKILL.md](../../../skills/exam-cram/SKILL.md)（控制层，单一事实源）。

## Student-facing Output

`中文` 模式（及 `双语` 的 `zh` 侧）学生侧使用规范中文词表（当前阶段 / 这题考什么 / 标准答题步骤 / 易错点 / 3分钟速记 / 现在轮到你 / 已记录到错题本 / 必背 / 例题 / 例题解答 / 要点解释 / 错题重做 / 疑难复述 / 已初始化备考空间）；`English` 模式使用英文规范词表（派发规则见控制层 `Output Contract` 与 [`docs/language-policy.md`](../../../docs/language-policy.md)）。`中文` 模式下来源标注逐字呈现为：

- 🟢 **来自资料**：直接源自学生上传内容，可信度高。
- 🟡 **AI 补充**：资料未覆盖、AI 用自身知识补的，标注「🟡 AI补充，可能与你老师讲的不完全一致」（以老师为准）。
- ⚠️ **AI 生成答案**：老师只勾题没给答案时 AI 代答的，每个都标「⚠️ AI生成答案，非老师/教材提供」。

学生可见输出默认英文（学生用中文开场则简体中文）；已持久化的 `language` 按控制层 `Output Contract` 派发规则切换，各模式单语言纯净。

双语组合规则（`language=bilingual`，显示别名为 `双语`）：绝不使用第三套模板——逐块组合：先出 `zh` 块（纯中文、中文规范形），紧跟一行 `> EN:` 英文镜像（纯英文、英文规范词表）；两侧各自单语言纯净、每个锚点每侧只出现一次。进度面板、回执、来源块也逐行同法镜像。`≤1天` 档可以缩短两侧措辞，但任何双语块都不能缺失其中一种语言；持久化的笔记本与章节教材始终保持完整中英双语。

**先落盘、再给摘要**：实质性回答（七步精讲、判分反馈、疑难解答、复盘结论，含随口的概念问答）默认先写进工作区笔记本 `notebook/`（错题同步镜像 `mistakes/`），聊天里只给 3–5 行摘要加一行完整链接（如 完整解答：`notebook/ch02.md#q13`｜目录：`notebook/index.md`）；只有进度面板、速查卡、一次性逃生提示可以只走聊天。纯网页端（无法写文件）保持原有纯聊天＋文本断点模式。落盘失败会明说，并把完整内容直接贴在聊天里。

**工作区落点**：首次激活先看注册表——没登记过课程就先问「材料文件夹在哪」并给 30 秒上手演示；登记过就问「继续哪门课」。绝不在你不知道的地方悄悄建文件夹；每次开场进度面板固定带一行工作区绝对路径，你永远知道自己的文件在哪。
