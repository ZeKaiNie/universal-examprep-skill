# 工作区审计说明

中文 · [English](../skills/exam-audit/SKILL.md)

`exam-audit` 用来只读检查一个已经生成的备考工作区。它适合排查缺少章节、题目无法判分、图片或答案遗漏、材料更新后产物过期、学习进度对不上等问题。

审计默认只报告问题，不修改、不删除文件。需要修复时，应先让学生看完问题清单并明确同意，再逐项处理或交回 `exam-ingest` 重建。

## 怎样使用

可以直接对智能体说：

```text
请用 exam-audit 只读检查这个备考工作区，列出所有阻断、警告和提示。不要自动修改任何文件。
```

智能体还应运行项目的静态校验器：

```bash
python scripts/validate_workspace.py <工作区路径> --json
```

命令中的 `ok=true` 只说明工作区没有结构性错误，不代表所有材料已经识别完整，也不代表没有警告。

## 审计会检查什么

1. **章节结构。** `study_plan.md` 里的每个阶段是否有对应的 `references/wiki/chN_*.md`，是否存在孤立章节或失效链接。
2. **标准题库。** `references/quiz_bank.json` 中的题型、选项、答案、关键词和来源字段是否完整；缺少官方答案的问题是否被诚实标记并进入待审流程。
3. **来源诚实性。** AI 补充和 AI 生成答案是否使用了正确标签，是否有人把它们误写成老师或教材提供的内容。
4. **计划与进度。** `study_state.json`、生成视图 `study_progress.md`、错题记录和阶段清单是否一致。存在 `study_state.json` 时，它才是进度事实源。
5. **教学例题保留。** 原本识别出的教学例题是否在重新 ingestion 后丢失，章节、来源页、答案来源和图片路径是否仍然有效。
6. **图片完整性。** 分别检查知识页、题面和答案三侧的图片证据；某一侧没有问题，不代表另外两侧完整。
7. **结构化 ingestion。** `.ingest/` 里的来源、内容单元、待审问题、补丁、构建清单和哈希是否互相一致，原材料是否已经发生漂移。
8. **章节完成证据。** 已勾选章节是否真的具备讲解、笔记、题库和复习讲义等当前模式要求的证据；“文件存在”本身不能证明已经完成。
9. **路径与隐私安全。** 是否存在目录穿越、符号链接逃逸、错误的绝对路径、学生作答图片被当成官方题面或答案等问题。

## 怎样读结论

- `ready`：没有当前错误或警告；它仍然不是“语义上绝对没有漏识别”的证明。
- `usable_with_gaps`：没有阻断错误，但仍有明确警告或证据缺口；开始学习前应先告诉学生这些限制。
- `blocked`：存在至少一个阻断问题，不能继续教学、测验或声称章节完成，应先回到 ingestion 或待审流程。

每条问题都应写出严重程度、具体文件、可观察到的现象和建议修法。无法恢复的内容也必须保留在报告里并说明原因，不能静默丢弃。

## 边界

- 审计不是重建工具，默认零写入、零删除。
- 不猜老师意图，只报告能够由文件和来源证据确认的不一致。
- 不把 AI 生成内容冒充标准答案，也不因为视觉索引没有报警就声称所有图片都已经找全。
- 不手工修改结构化事实文件来绕过校验器。

完整、可执行的行为契约仍以 [`skills/exam-audit/SKILL.md`](../skills/exam-audit/SKILL.md) 为准；本页是给中文读者看的说明。
