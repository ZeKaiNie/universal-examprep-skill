# Feature × Tier Coverage Matrix

English · [中文](coverage-matrix.md)

> Companion to [`testing-audit.md`](testing-audit.md). In one sentence: **many capabilities are tested statically at the structure/instruction layer but have not been executed and verified at the behavior layer.**

**Legend**: ✅ covered　🟡 partial (structure/schema layer, **or** Tier 2 deterministic mock detection—not real LLM behavior)　❌ not covered　— not applicable

> **Tier 2 column note**: 🟡 means this PR’s (T2) **deterministic mock detector** covers artifact assertions for the behavior (in CI, zero cost). It **does not mean** real LLM behavior has been verified. Real-Agent smoke is opt-in and excluded from CI; see [`../behavior_smoke/`](../behavior_smoke/).
> **B2 is wired**: `behavior_smoke --llm` single-turn real-Agent smoke is no longer a skeleton. It drives `claude -p` per scenario (or an `--agent-cmd` stub), applies the **same detector function** as `--mock` to the response (the primary positive check for each scenario; mock-only extra negative/variant assertions validate the detector and are not rerun live), writes a transcript, and emits metrics. State/file scenarios that a one-shot `-p` cannot verify are honestly reported as SKIP. This is opt-in, off by default, and excluded from CI; `tests/test_behavior_smoke_live.py` verifies the wiring deterministically with a stub Agent. Every table note saying “real LLM … unverified (opt-in)” refers to this route.
>
> **Tier 4 column note**: 🟡 means PR T4’s **deterministic replay harness** in [`../drift/`](../drift/) covers the behavior across **multi-turn, long-running conversations** by replaying scripted transcripts plus snapshots in root tests at zero cost. It likewise **does not mean** real LLM long-session behavior has been verified. Real-Agent long sessions are opt-in, unimplemented, and excluded from CI.

## How to read this table

- **Many ✅/🟡 entries in the first two columns (Tier 0 / Tier 1)**: structure, schemas, provenance rules, path safety, and ingestion are tested at zero cost.
- **Existing Tier 2 deterministic mock layer (🟡)**: `scenarios.json` contains **17 behavior scenarios**. **16 are asserted in deterministic `--mock`, run in CI, and cost nothing**. One (`lazy_load_best_effort`) is marked `best_effort:true` and reports `SKIP` without an assertion in deterministic mode because it requires a transcript/real LLM. The 16 asserted scenarios comprise the original seven (`quiz_bank`-only / provenance labels / hint-skip-mistake archive / confusion tracking / checkpoint recovery / no-Python fallback / beginner walkthrough upgraded to seven steps), five added by line A (`visual_first_assets` prompt-image gate / `scope_override` override announcement / `teaching_template` seven-step template + source block / `time_budget_no_questions` ≤1-day cadence and explicit no-questions split / `knowledge_window_recheck` out-of-window recheck), one added by A8b (`language_first_ask` combined language question), two v4 red lines (`notebook_persist_ok` teaching persistence receipt / `workspace_confirm_ok` workspace confirmation), and one artifact-resource gate (`artifact_mode_routing`). See [`../behavior_smoke/README.md`](../behavior_smoke/README.md) for the inventory.
- **New Tier 4 deterministic replay layer (🟡)**: [`../drift/`](../drift/) covers goal retention, plan adherence, question-invention rate, checkpoint recovery, provenance-label fidelity, progress persistence, and cross-chapter wiki reads across **multi-turn long sessions**. It also covers A6’s `mode_urgent_no_questions` (zero interactive questions after an explicit no-questions request + urgent-opening inference and persistence) and B3’s `window_persist` (long-session knowledge-window persistence through `window_rows_added`/`window_rows_lost`). It replays scripted transcripts plus snapshots in root tests at zero cost.
- **B3: Tier 4 real-Agent route (opt-in)**: `run_drift.py --llm` is no longer a skeleton. It delegates to [`../drift/run_live_smoke.py`](../drift/run_live_smoke.py), which drives real-Agent long sessions with token limits, fail-fast termination, and a fixture sandbox → records T5b logs → converts them to JSONL → grades with the **same** `compute_metrics`/thresholds → records the run. It requires `RUN_SKILL_DRIFT_LLM=1`, incurs real invocation cost, and **never runs in CI**.
- **B1 final-scan principle**: Every fixed defect on line A has a corresponding deterministic test that **turns red**—either a Tier 0 unit test or a negative assertion in a Tier 2 mock detector. This table records each location. Real-LLM behavior remains opt-in and excluded from CI.
- **But 🟡 ≠ ✅**: The deterministic layers (Tier 2 mock and Tier 4 replay) prove only that detectors work on expected artifacts/scripted conversations. They **do not prove that a real LLM Agent will necessarily produce those behaviors**. Real behavior coverage requires the opt-in `--llm` route, which is off by default, unimplemented, and excluded from CI. Lazy loading and compute-before-drawing remain best-effort/uncovered.
- **Most gaps are best filled with deterministic structural assertions** over artifact files and do not require an LLM judge. See [`testing-audit.md`](testing-audit.md) §9 and [`../behavior_smoke/README.md`](../behavior_smoke/README.md).
