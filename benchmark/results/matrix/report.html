装了这个备考 skill，AI 真的更少「胡编」吗？——MIT 6.006 实测

测法：同一套金标题（65 题，全部锚定 MIT 6.006 官方讲义/习题），让 3 个模型在 3 种条件下作答——

  • 闭卷：不给任何材料，只靠模型自己的知识。
  • 给全材料：把整门课全文塞进提示里（dump）。
  • skill：只把课件建成知识库，模型按需检索（skill 的防幻觉机制）。
一句话结论：在最弱的 Haiku 上，闭卷正确率 11% → 给全材料 4% → skill 惰性检索 38%；越界题上 skill 的弃答率 100%（闭卷只有 60%，即闭卷在不会的题上更爱硬编）。

⚠️ 诚实声明 Caveats

  • 题量 n=65（每条都跨 3 臂同题对比，最公平）。
  • 裁判＝sonnet（judge_repeats=1），数值题为确定性判分。裁判与被测同属一个模型家族，仅做了跨模型而非人工 kappa 校准——属已知局限。

Does the exam-prep skill cut hallucination? A MIT 6.006 benchmark

Method: one gold set (65 items, all grounded in official MIT 6.006 lecture notes / problem sets) answered by 3 models under 3 conditions —

  • Closed-book：no materials, parametric knowledge only.
  • Full-materials：the entire course text dumped into the prompt.
  • Skill：course built into a wiki the model retrieves from on demand (the skill's regime).
Bottom line：On the weakest model (Haiku): closed-book correctness 11% → full-materials 4% → skill retrieval 38%; on out-of-scope probes the skill abstains 100% vs closed-book 60% (closed-book fabricates more when it doesn't know).

🧪 Hallucination & out-of-scope abstention

Hallucination = share of answers with claims not supported by (or contradicting) the source (lower is better; judged against the full lecture, so it penalizes correct-but-unsourced elaboration). OOS abstention = share of not-covered probes where the model honestly says “not covered” (higher is better).

📈 Convergence: more wiki coverage → more correct

Skill arm over 3 rounds as the wiki grows 7 → 14 → 20 chapters (lecture items + probes, Haiku).

⚠️ Caveats

  • n=65 items, each answered under all conditions.
  • Judge = sonnet (repeats=1); numeric items scored deterministically. Judge and tested models share one family; only cross-model (not human-kappa) calibration was done — a known limitation.
  • Faithfulness is judged against the full lecture, so correct-but-unsourced elaboration counts as unfaithful — a reasonable grounding criterion, but worth knowing.
  • This report concerns the skill itself only, independent of any future platform. Data and code are reproducible.
